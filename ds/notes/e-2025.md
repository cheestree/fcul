#### 2

a) Availability is the degree to which a system is operational and accessible when required for use. It is often expressed as a percentage of uptime over a specified period. Reliability however is the degree to which a system performs its intended functions without failure over a specified period under specified conditions.

b)

R of DB2 = R2
R of the subsystem = R1 + (1 - R1) * R2

R of the system = Rs * R of the subsystem

R of the system = Rs *[R1 + (1 - R1)* R2]

### G2

#### 1

a) A pipe-filter architecture could be used, as it allows for data to be processed in a series of independent stages (filters) connected by data streams (pipes). Each filter can perform a specific transformation on the data, making it easy to add, remove, or modify processing steps without affecting the overall system. It can be expanded by parallelizing filters or adding new filters to the pipeline. Some filters can be used for virus scanning, others for link analysis, and others for content extraction. The final filter decides whether to store or discard the email based on the results of the previous filters, and an extra filter handles user notifications.

b)

#### 2

The use of pipe interfaces allows for easy integration of new filters into the existing architecture. Each filter operates independently and communicates with others through well-defined interfaces (pipes). This modularity enables developers to add new functionality, such as additional email processing steps, without needing to modify the existing filters. The system can be easily extended by inserting new filters into the pipeline, allowing for scalability and adaptability to changing requirements.

#### 3

An extra filter/pipe can be added to handle phishing attempt filtering by simply inserting it between the existing filters, without affecting the other components. This filter would analyze the email content for common phishing indicators and either flag or discard suspicious emails before they reach the storage or notification stages. In other architectures, such as monolithic or tightly coupled systems, adding new functionality would require significant changes to the existing codebase, potentially introducing bugs and increasing maintenance complexity.
