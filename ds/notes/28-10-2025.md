#### Broker

Usually implemented in situations where there are components on different platforms that need to interoperate.  
Remote service invocation is done via the broker.  

Elements:

- Broker - coordinates communications between distributed.
- Server
- Client
- Service invocation
  
Advantages:

- Location transparency
- Changeability
- Portability

Disadvantages:

- Performance
- Single point of failure
- Target for security attacks
- Challenging to test
