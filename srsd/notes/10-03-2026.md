# RSA algorithm

Based on the correspondence of a block to a number, both keys are pairs of numbers. The encryption is done by raising the plaintext to the power of the public key exponent and taking the modulus of the product of the two prime numbers. The decryption is done by raising the ciphertext to the power of the private key exponent and taking the modulus of the product of the two prime numbers.

It uses the Euler theorem to ensure that the decryption works correctly. The security of RSA relies on the difficulty of factoring large composite numbers, which is a hard problem in number theory. The key generation process involves selecting two large prime numbers, calculating their product, and then determining the public and private key exponents based on the Euler totient function. RSA is widely used for secure communication and digital signatures.

## Attacks on RSA

Brute force attacks - can be employed, but the key size must be sufficiently large to make it computationally infeasible. For example, a key size of 2048 bits is considered secure against brute-force attacks with current technology. Solution: larger keys, performance trade-off.

Mathematical attacks - such as factoring the modulus or finding the private key exponent. These attacks can be mitigated by using larger key sizes and ensuring that the prime numbers used in key generation are sufficiently large and random.

Timing attacks - can be used to extract information about the private key by measuring the time taken for encryption or decryption operations. Countermeasures include implementing constant-time algorithms and adding random delays to the operations.

# Hashing algorithm

A hashing algorithm takes an input (or message) and produces a fixed-size string of bytes, typically a digest that is unique to the input. The output is often referred to as a hash value or hash code. Hashing algorithms are designed to be fast and efficient, and they are commonly used for data integrity verification, password storage, and digital signatures.

Given an arbitrary input, it is easy to compute the hashed value. However, it is computationally infeasible to reverse the process and obtain the original input from the hash value (pre-image resistance). Given a different input, it is also computationally infeasible to produce the same hash value (collision resistance). Additionally, a small change in the input should produce a significantly different hash value (avalanche effect).

## Attacks on hashing algorithms

Brute-force attacks - can be used to find an input that produces a specific hash value, but the key space is typically very large, making it computationally infeasible. For example, a hash function with a 128-bit output has 2^128 possible hash values, which is astronomically large.

Cryptanalysis attacks - can be used to find weaknesses in the hash function that allow for collisions or pre-image attacks. For example, if a hash function has a known weakness that allows for finding two different inputs that produce the same hash value (a collision), then it is not secure.

## Birthday paradox

The birthday paradox is a counterintuitive probability problem that states that in a group of just 23 people, there is a 50% chance that at least two people share the same birthday. This paradox arises because the number of possible pairs of people increases rapidly as the group size increases. In the context of hashing algorithms, the birthday paradox implies that if a hash function produces a fixed-size output (e.g., 128 bits), then after hashing approximately 2^64 different inputs, there is a significant probability of finding two different inputs that produce the same hash value (a collision). This is why it is important to use hash functions with sufficiently large output sizes to mitigate the risk of collisions.

## Building a hash algorithm

By dividing the message in blocks and applying a compression function iteratively, we can create a hash algorithm. The compression function takes a fixed-size input and produces a fixed-size output, which is then used as input for the next iteration along with the next block of the message. This process continues until all blocks of the message have been processed, resulting in the final hash value.
