# Segurança de Redes e Sistemas Distribuídos

## Cryptoanalysis

Cryptoanalysis - based on the nature of the algorithm and knowledge of the plaintext (statistical analysis, frequency analysis, etc.). Either gets the plaintett or the key. Can be easier based on the amount of info known.

Unconditionally secure - no amount of computational power can break the cipher, as the generated ciphertext does not have enough information to determine uniquely the corresponding plaintext. One-time pad is an example, but it has practical issues (key distribution, key management, etc.).

Computationally secure - either (or both) of the following holds:

- the cost of breaking the cipher is more than the value of the protected information, or
- the time required to break the cipher is more than the useful lifetime of the protected information.

## Brute-force

If it is assumed that the algorithm is secure, then the only way to break it is through brute-force, which is trying all possible keys until the correct one is found.

Attacks on the key with known-plaintext:

- one knows a few pairs of plaintexts and the corresponding ciphertexts.
- keys are tried (even all possible keys) until a key is found that produces the correct ciphertext for the known plaintext.
- the number of bits in the key determines the security of the cipher against brute-force attacks. For example, a 128-bit key has 2^128 possible keys, which is computationally infeasible to break with current technology.
