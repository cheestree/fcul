# Segurança de Redes e Sistemas Distribuídos

## Cryptoanalysis

Cryptoanalysis - based on the nature of the algorithm and knowledge of the plaintext (statistical analysis, frequency analysis, etc.). Either gets the plaintett or the key. Can be easier based on the amount of info known.

Unconditionally secure - no amount of computational power can break the cipher, as the generated ciphertext does not have enough information to determine uniquely the corresponding plaintext. One-time pad is an example, but it has practical issues (key distribution, key management, etc.).

Computationally secure - either (or both) of the following holds:

- the cost of breaking the cipher is more than the value of the protected information, or
- the time required to break the cipher is more than the useful lifetime of the protected information.

## Brute-force

If it is assumed that the algorithm is secure, then the only way to break it is through brute-force, which is trying all possible keys until the correct one is found.

Attacks on the key with known-plaintext:

- one knows a few pairs of plaintexts and the corresponding ciphertexts.
- keys are tried (even all possible keys) until a key is found that produces the correct ciphertext for the known plaintext.
- the number of bits in the key determines the security of the cipher against brute-force attacks. For example, a 128-bit key has 2^128 possible keys, which is computationally infeasible to break with current technology.

## Feistel Structure

A Feistel structure is a design model for building block ciphers. It divides the input block into two halves and processes them through multiple rounds of encryption. Each round consists of a function that takes one half of the data and combines it with the other half using a key. The output of each round is then used as input for the next round. This structure allows for the creation of strong encryption algorithms while maintaining efficiency and security. Examples of ciphers that use Feistel structures include DES (Data Encryption Standard) and Blowfish.

## General Principles for the Design

Number of rounds - more rounds generally increase security, but also increase computational cost. A common choice is 16 rounds, as in DES.
Design of function F - should be complex and non-linear to prevent attacks. It should also be designed to ensure that small changes in the input produce significant changes in the output (avalanche effect).
Key schedule - the method of generating round keys from the main key should be secure and should not produce weak keys.
Block size - larger block sizes can provide better security, but also increase computational cost. Common block sizes are 64 bits (as in DES) and 128 bits (as in AES).

## Variants

### SM4

SM4 is a block cipher that uses a Feistel structure. It has a block size of 128 bits and a key size of 128 bits. It consists of 32 rounds of encryption, with a complex function F that includes substitution and permutation operations. The key schedule is designed to generate round keys that are derived from the main key in a secure manner. SM4 is widely used in China for secure communication and data protection.

### ARIA

ARIA is a block cipher that also uses a Feistel structure. It has a block size of 128 bits and supports key sizes of 128, 192, and 256 bits. ARIA consists of 12 rounds for 128-bit keys, 14 rounds for 192-bit keys, and 16 rounds for 256-bit keys. The function F in ARIA includes substitution and permutation operations, as well as a key mixing step. The key schedule is designed to generate round keys that are derived from the main key in a secure manner. ARIA is used in South Korea for secure communication and data protection.

## Cipher Modes

ECB (Electronic Codebook) - the simplest mode, where each block of plaintext is encrypted independently. However, it is not recommended for secure encryption as it can reveal patterns in the plaintext.
CBC (Cipher Block Chaining) - each block of plaintext is XORed with the previous ciphertext block before being encrypted. This mode provides better security than ECB, as it hides patterns in the plaintext.
CFB (Cipher Feedback) - similar to CBC, but it allows for encryption of data in smaller units than the block size. It also provides better security than ECB.
OFB (Output Feedback) - similar to CFB, but it generates keystream blocks that are XORed with the plaintext blocks to produce the ciphertext. It also provides better security than ECB.
CTR (Counter) - generates a keystream by encrypting a counter value that is incremented for each block. The keystream is then XORed with the plaintext to produce the ciphertext. This mode allows for parallel encryption and decryption, and it provides better security than ECB.
