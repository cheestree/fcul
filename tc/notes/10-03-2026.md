# Técnicas de Compilação

A lexer is responsible for breaking down the source code into tokens, which are the basic building blocks of a programming language. It removes unnecessary whitespace and comments, and identifies keywords, identifiers, literals, and operators.

For example:

- Keywords: `if`, `else`, `while`, `for`
- Identifiers: `myVariable`, `calculateSum`
- Literals: `42`, `"Hello, World!"`
- Operators: `+`, `-`, `*`, `/`
- Comments: `// This is a comment`

The parser takes the tokens produced by the lexer and builds a parse tree (or abstract syntax tree) based on the grammar of the programming language. It checks for syntax errors and ensures that the code follows the correct structure.

A language is a set of strings that are valid according to a specific grammar. The grammar defines the rules for how tokens can be combined to form valid statements and expressions in the programming language.

A grammar is composed by a set of production rules, which define how non-terminal symbols can be replaced by sequences of terminal and non-terminal symbols. For example, a simple grammar for arithmetic expressions might include rules like:

```expr -> expr + term
expr -> expr - term
expr -> term
term -> term * factor
term -> term / factor
term -> factor
factor -> ( expr )
factor -> number
```

Regex rules are as follows:

- 'a' matches the character 'a'
- 'a*' matches zero or more occurrences of 'a'
- 'a+' matches one or more occurrences of 'a'
- 'a?' matches zero or one occurrence of 'a'
- 'a|b' matches either 'a' or 'b'
- 'a.b' matches 'a' followed by any character followed by 'b'
- 'epsilon' matches the empty string
- '[abc]' matches any one of the characters 'a', 'b', or 'c'
- '[a-z]' matches any lowercase letter from 'a' to 'z'
- '[A-Z]' matches any uppercase letter from 'A' to 'Z'
- '[0-9]' matches any digit from '0' to '9'
- '\d' matches any digit (equivalent to '[0-9]')
- '\w' matches any word character (equivalent to '[a-zA-Z0-9_]')
- '\s' matches any whitespace character (spaces, tabs, etc.)
- '.' matches any character except a newline
- '^' matches the start of a string
- '$' matches the end of a string

A DFA is a finite state machine that accepts or rejects strings of symbols and can be used to recognize patterns defined by regular expressions. It consists of a finite number of states, including a start state and one or more accept states. The DFA transitions between states based on the input symbols it receives.

A NDFA is a finite state machine that can be in multiple states at once. It allows for multiple transitions for the same input symbol from a given state, and it can also include epsilon transitions (transitions that occur without consuming any input). An NDFA can be converted to an equivalent DFA using the subset construction algorithm.

The order of grammar rules matter because it can affect the way the parser constructs the parse tree. If there are multiple rules that can match a given input, the parser will typically choose the first rule that matches. This can lead to different parse trees and potentially different interpretations of the code. Therefore, it is important to carefully order the grammar rules to ensure that the parser behaves as intended.
