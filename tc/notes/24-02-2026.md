# Técnicas de Compilação

Lexer - lexical analyser that takes source code and breaks it into tokens. Removes whitespaces and comments. It also identifies keywords, identifiers, literals, and operators.
Parser - syntactic analyser that takes the tokens produced by the lexer and builds a parse tree (or abstract syntax tree) based on the grammar of the programming language. It checks for syntax errors and ensures that the code follows the correct structure.
Validator - semantic analyser that checks the parse tree for semantic errors, such as type mismatches, undeclared variables, and scope issues. It ensures that the code is semantically correct and can be executed. Can either go directly to code generation, or an intermediate representation, or exit with an error message.
Code Generator - takes the validated parse tree and generates target code, which can be machine code, assembly code, or bytecode, depending on the target platform. It translates the high-level constructs of the source code into low-level instructions that can be executed by the computer. It may also perform optimizations to improve the performance of the generated code.
